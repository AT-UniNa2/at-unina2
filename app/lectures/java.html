A gently introduction to Java

History


Software for early computers was primarily written in assembly language.

An assembly language is a low-level programming language for a computer in which there is a one-to-one correspondence between the language and the architecture's machine code instructions.

Assembly language is converted into executable machine code by an assembler.

Assembly example


            
move $v0, $a0					
beqz $a1, gcd_return
move $t4, $a0
move $a0, $a1
remu $a1, $t4, $a1
jal gcd
lw $a1, 8($sp)
lw $a0, 4($sp)

The assembler reads the source code row-by-row and replaces each label with the corresponding binary code.

Programming in assembly


Each architecture has its own assembly language!

Introducing the compilers


With the constant hardware producers growth, it became necessary to contrive a way to reuse the same software on different kinds of CPUs.

Towards the end of the 1950s the first compilers were developed making the way straight for the high-level programming languages.

Using the compiler


The source code is written once and compiled for each architecture.

Introducing the Java Virtual Machine


The compiler introduction resolves the multiple implementation issue.

Anyway the programmers still need to compile their source code for each architecture.

Java programming language removes this burden by introducing the Java Virtual Machine.

Introducing the Java Virtual Machine


The JVM is a an abstract computing machine, running as a process over a real machine, that execute any program compiled into Java bytecode.

Using the java virtual machine


The source code is compiled once and executed in a virtual machine on each architecture.

The JVM process


The Java Language


Java is a concurrent, object-oriented programming language.

It is one of the most popular programming language in use, particulary for client-server web applications.

The language derives much of its syntax from C and C++.

Java is object oriented


Five are the concept to keep in mind to apply a pure object oriented programming approach:

  • everything is an object;
  • a program is a bunch of objects telling each other what to do by sending messages;
  • each object has its own memory made by other objects;
  • every object has a type;
  • all object of a particular type can receive the same messages.

Every object has an interface


An interface defines the set of requests you can make to an object.

Using this approach an object can be used as a service provider.

Defining an interface


implicitly

                
public class Lightbulb {
  public void on() {
    // implement here
  }
  public void off() {
    // implement here
  }
}

explicitly

                
public interface Lightbulb {
  public void on();
  public void off();
}

The has-a relationship


Every class represent an useful unit of code that can be reused.

The classes reuse is achieved by including an object of a class into the new class.

This pattern in known as composition.

Encapsulating data


            
public class Car {
  private Engine engine;
  
  public Car() {
    this.engine = new Engine();
  }
  
  public void start() {
    this.engine.turnOn();
  }
}

The is-a relationship


Two types can have characteristics and behaviors in common, but one type may handle more messages or handle them differently.

Inheritance expresses this similarity between types by using the concept of base types and derived types.

A base type contains all the characteristics that are shared among the derived types.

Applying the inheritance


            
public abstract class Shape {
  public Shape() {
    System.out.println("Creating a shape");
  }
  
  public abstract void draw();
}
            
          
            
public class Circle extends Shape {
  public Circle() {
    super();
  }
  
  @Override
  public void draw() {
    // circle implementation
  }
}
