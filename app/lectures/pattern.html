Mobile constraints


  • The network is unreliable;
  • File sizes need to be small;
  • Debugging requires logging;
  • Minimize user input;
  • Minimize time to result.

Representational state transfer


REST is a simple stateless architecture that generally runs over HTTP.

It is based on the Hypermedia as the Engine of Application State (HATEOAS) principle.

Hypermedia as the Engine of Application State


A client interacts with a network application entirely through hypermedia (URLs) provided dynamically by application servers.

Client and server don't need to share an interface anymore.

This allows the server functionalities to evolve independently.

REST in web services


RESTful APIs leverage on:

  • internet media types (JSON, XML, images...)
  • HTTP methods (GET, POST, PUT, DELETE)
  • hyperlinks to reference resources (http://example.com/item)

REST fundamentals


  • Nouns to define URLs
  • Two types of data:
    • Resources
    • Collections (Set of resources)
    Data are represented in XML or JSON format.

REST fundamentals


Data operations are mapped 1:1 over HTTP methods:

PUT => Create
GET => Read
POST => Update
DELETE => Delete

PUT and POST can both be used for Create and Update
