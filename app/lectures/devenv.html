Generating a Vagrantfile


The Vagrantfile is used to configure the development linux box we will run.

            
$ vagrant init at-box https://cloud-images.ubuntu.com/vagrant/trusty/current/trusty-server-cloudimg-i386-vagrant-disk1.box
A `Vagrantfile` has been placed in this directory. You are now
ready to `vagrant up` your first virtual environment! Please read
the comments in the Vagrantfile as well as documentation on
`vagrantup.com` for more information on using Vagrant.
vagrant init at-box   1.42s user 0.24s system 84% cpu 1.962 total

$ ls
Vagrantfile
  • at-box is the box local name
  • we will download the Ubuntu Server 14.04.1 LTS box

Vagrantfile


            
VAGRANTFILE_API_VERSION = "2"

Vagrant.configure(VAGRANTFILE_API_VERSION) do |config|
  # The name we chose at initialization time.
  # Vagrant files having same box name will use the same box.
  config.vm.box = "at-box"

  # The URL where the box is fetched.
  config.vm.box_url = "https://cloud-images.ubuntu.com/vagrant/trusty/current/trusty-server-cloudimg-i386-vagrant-disk1.box"

  # With this option you enable the forward of connections from real machine to the currently running box.
  # config.vm.network "forwarded_port", guest: 80, host: 8080

  # This option adds a new shared folder to the box.
  # The current folder is already mounted in the box as /vagrant folder.
  # config.vm.synced_folder "../data", "/vagrant_data"
end

Install Google Chrome Canary



Go to https://www.google.com/chrome/browser/ and download Google Chrome for your operating system

Why Canary?


  • features not available in Chrome yet;
  • private user configurations (not shared with Chrome);
  • advanced dev tools;
  • mobile device simulator.
